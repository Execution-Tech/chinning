## Implementation Steps

### Step 1 — Fix `src/utils/index.js` auth methods

1. Add `verifyToken` method: `POST /auth/verify-token` with `{phone, token}`
2. Remove/replace `verifyAccount` (wrong URL `/auth/verify-account` → not used after this)
3. Remove/replace `resendVerificationAccount` (wrong URL `/auth/resend-verify-account` → not used)
4. Keep `verifyCode` as-is (correct URL `/auth/verify-code`)
5. Keep `resendVerificationCode` as-is (correct URL `/auth/resend-code`)
6. Fix baseURL fallback from `yousr.mangjornal.com` → `test-back.yosrtech.com/api/v1/`

### Step 2 — Rewrite `src/context/AuthContext.js`

Replace all 4 stub methods with real API calls:

**`login(credentials)`**:
- Call `ecommerceAPI.auth.login({ phone, password, fcm_token: "", lang: getLocale(), type: "web" })`
- On success: extract `token` + `user` from response, call `ecommerceAPI.setAuthToken(token)`, dispatch `LOGIN_SUCCESS`
- On error: dispatch `SET_LOADING(false)`, `toast.error(err.data?.message || err.message)`

**`register(userData)`**:
- Call `ecommerceAPI.auth.register({ name, phone, password, password_confirmation })`
- On success: redirect to `/${locale}/verification-code?identifier=${phone}&flow=registration`
- On error: toast error

**`logout()`**:
- Call `ecommerceAPI.auth.logout()` (wrapped in try/catch — always clear regardless)
- Call `ecommerceAPI.clearAuth()`
- Dispatch `LOGOUT_SUCCESS`
- Redirect to `/${getLocale()}`

**`initAuth()` (NEW)**:
- Read `localStorage.getItem("access_token")`
- If token exists: call `ecommerceAPI.auth.getProfile()` → dispatch `LOGIN_SUCCESS` with user
- If no token or 401: dispatch `LOGOUT_SUCCESS`
- Call in `useEffect([], [])` inside `AuthProvider`

**`updateProfile(userData)`**:
- Call `ecommerceAPI.auth.updateProfile(userData)` (or keep as local dispatch if endpoint not used yet)
- Dispatch `UPDATE_USER`

Replace the stub `handleSubmit` with:

```
if (response.data?.status) {

  toast.error(response.data?.message || "فشل إرسال الرمز")
}
```
- Add `loading` state + disable button while loading
- Import `ecommerceAPI` from `@/utils`
- Import `useRouter` from `next/navigation`

### Step 4 — Rewrite `src/app/[locale]/verification-code/page.tsx`

Read URL params `flow`, `phone`, `identifier` via `useSearchParams()`.

**On OTP complete (`handleComplete(code)`):**
```
if (flow === "registration") {
  → call ecommerceAPI.auth.verifyCode(code, identifier)
  → on success: router.push(`/${locale}`)  // home, user now authenticated
} else if (flow === "forgot-password") {
  → call ecommerceAPI.auth.verifyToken(phone, code)
  → on success: router.push(`/${locale}/create-new-password?phone=${phone}&token=${code}`)
}
```

**Resend button (`handleResend`):**
```
if (flow === "registration") → ecommerceAPI.auth.resendVerificationCode(identifier)
if (flow === "forgot-password") → ecommerceAPI.auth.forgotPassword(phone)
```

- Remove hardcoded `"1234"` check
- Add `loading` state during API call
